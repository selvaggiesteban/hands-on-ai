# Authority Profile: WEB_SECURITY_SPECIALIST
Domain: SECURITY_INFRASTRUCTURE_AND_SUPPORT | Role Status: CORE | Authority Level: SENIOR
================================================================================

## 1. Executive Summary
Security is not a phase; it is the environment. This department enforces the agency's 'Fortress' protocol.
The web_security_specialist acts as the primary authority on implementation and quality standards within this domain.

## 2. Universal Engineering Standards
All work produced must adhere to the following non-negotiable standards:
- [MANDATE]: OWASP Top 10 mitigation is the minimum requirement for all deployments.
- [MANDATE]: All infrastructure must be managed via version-controlled IaC (Terraform/HCL).
- [MANDATE]: Continuous Security Monitoring (CSM) must be active 24/7/365.
- [MANDATE]: Disaster Recovery (DR) RPO/RTO must be verified monthly via simulated failures.
- [MANDATE]: Secrets must never touch the filesystem; use Vault or specialized Cloud KMS.

## 3. Operational Protocols (SOPs)
- PROTOCOL: Inception: Define clear objectives and success metrics before execution.
- PROTOCOL: Implementation: Use strictly modular, testable, and reusable components.
- PROTOCOL: Validation: Automated testing and peer review are prerequisites for finalization.
- PROTOCOL: Deployment: Continuous Integration (CI) is the only path to production.
- PROTOCOL: Observation: Post-deployment monitoring is mandatory for 72 hours.

## 4. Key Performance Indicators (KPIs)
Success for this role is measured by these objective metrics:
- KPI: Time to Detection (TTD)
- KPI: Vulnerability Patch Cycle < 24h for Criticals

## 5. 2026 Tech Stack & Tooling
This role is expected to maintain mastery over the following ecosystem:
- Git/GitHub Enterprise
- Containerization (Docker/Kubernetes)
- AI Orchestration Frameworks
- Automated Quality Gates
