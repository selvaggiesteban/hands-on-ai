# JWT

- Is a strong secret or private key used for signing? [YES/NO]
- Is the algorithm set explicitly to prevent 'None' algorithm attacks? [YES/NO]
- Are tokens short-lived (e.g., 15 minutes)? [YES/NO]
- Is a refresh token mechanism implemented? [YES/NO]
- Are sensitive data excluded from the payload? [YES/NO]
- Is the 'iss' (issuer) and 'aud' (audience) claim verified? [YES/NO]
