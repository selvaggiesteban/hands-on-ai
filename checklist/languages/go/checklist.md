# Go Standards

- Is the code formatted using 'gofmt'? [YES/NO]
- Are errors handled explicitly after every function call that returns one? [YES/NO]
- Is the project structured according to standard Go project layouts? [YES/NO]
- Are context objects used for cancellation and timeouts in networked services? [YES/NO]
- Are interfaces used to promote decoupling and testability? [YES/NO]
- Does the code pass 'go vet' and 'staticcheck'? [YES/NO]
