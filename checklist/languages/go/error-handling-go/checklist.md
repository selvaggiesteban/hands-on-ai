# Error Handling

- Are errors returned as the last return value? [YES/NO]
- Is 'errors.Is' used for comparing errors (Go 1.13+)? [YES/NO]
- Is 'errors.As' used for checking error types? [YES/NO]
- Are errors wrapped with '%w' when adding context? [YES/NO]
- Is the zero value of an error (nil) used to indicate success? [YES/NO]
- Are custom error types defined when specific handling logic is needed? [YES/NO]
