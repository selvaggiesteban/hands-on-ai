# ES6 Modules

- Are 'import' and 'export' statements used at the top level? [YES/NO]
- Are named exports preferred over default exports for refactoring safety? [YES/NO]
- Are file extensions included in imports if required (e.g., in browser native modules)? [YES/NO]
- Are circular dependencies avoided? [YES/NO]
- Is 'export *' avoided to prevent namespace pollution? [YES/NO]
- Are dynamic imports ('import()') used for code splitting? [YES/NO]
