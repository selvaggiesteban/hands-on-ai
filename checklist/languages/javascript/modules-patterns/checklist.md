# Modules and Patterns

- Are ES Modules (import/export) used instead of CommonJS where possible? [YES/NO]
- Is the Module Pattern used to encapsulate private data and expose public APIs? [YES/NO]
- Are circular dependencies identified and avoided? [YES/NO]
- Is 'default export' used sparingly in favor of 'named exports' for better tree-shaking? [YES/NO]
- Are side-effect only imports documented clearly? [YES/NO]
- Is the Singleton pattern used only when a single instance is strictly necessary? [YES/NO]
