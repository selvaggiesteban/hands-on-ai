# Lifetimes

- Are lifetime parameters ('a) named descriptively if complex? [YES/NO]
- Is lifetime elision relied upon for simple cases? [YES/NO]
- Are static lifetimes used only when necessary? [YES/NO]
- Are struct fields with references properly lifetime-annotated? [YES/NO]
- Is the relationship between input and output lifetimes clear? [YES/NO]
- Are lifetime issues resolved by design changes (owning data) rather than complex annotations? [YES/NO]
