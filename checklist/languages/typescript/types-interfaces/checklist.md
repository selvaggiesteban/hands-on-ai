# Types vs Interfaces

- Are 'interfaces' used for defining object shapes and public APIs (extensibility)? [YES/NO]
- Are 'type' aliases used for unions, primitives, and tuples? [YES/NO]
- Is consistency maintained (pick one strategy for objects)? [YES/NO]
- Are interfaces merged (declaration merging) where appropriate? [YES/NO]
- Are types used for mapped types and conditional types? [YES/NO]
- Is the performance difference (negligible) ignored in favor of semantics? [YES/NO]
