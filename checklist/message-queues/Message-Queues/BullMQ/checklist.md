# BullMQ

- Is Redis persistence configured if job durability is required? [YES/NO]
- Are failed jobs handled with a Dead Letter Queue (DLQ)? [YES/NO]
- Is concurrency managed to prevent overloading workers? [YES/NO]
- Are job data payloads kept small? [YES/NO]
- Is 'removeOnComplete' enabled to prevent Redis memory growth? [YES/NO]
- Are stalled jobs detected and recovered? [YES/NO]
