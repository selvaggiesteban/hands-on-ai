# General Testing Strategy

- Does the project have a clearly defined testing strategy (Unit, Integration, E2E)? [YES/NO]
- Is code coverage measured and monitored (e.g., > 80% target)? [YES/NO]
- Are unit tests isolated and fast-running? [YES/NO]
- Do integration tests verify interactions between components (e.g., Database, API)? [YES/NO]
- Are end-to-end tests automated for critical user journeys? [YES/NO]
- Is there a regression test suite run before every release? [YES/NO]
