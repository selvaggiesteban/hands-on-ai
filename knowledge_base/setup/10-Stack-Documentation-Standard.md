# Stack Documentation Standard

## 1. Required Sections (Mandatory)

Every technology stack README.md must include these sections in this exact order:

### 1.1 `# Name`
The official name of the technology.

### 1.2 `## Overview`
A brief description of what the technology is and its primary use case.

### 1.3 `## Stack / Tags`
Machine-readable list of associated technologies and concepts.
- Format: `[Tag1, Tag2, Tag3]`
- Example: `[Node.js, Express, API, Backend, JavaScript]`

### 1.4 `## Supported Operations`
List of actions the orchestrator can perform with this technology.
- Format: List of operation identifiers
- Example: `[create-route, add-middleware, generate-controller, run-tests]`

### 1.5 `## QA Checklist`
Quality assurance checks to be performed.
- Format: Markdown checklist
- Example:
  - [ ] Route follows RESTful principles
  - [ ] Middleware handles errors gracefully
  - [ ] Controller logic is separated from business logic

### 1.6 `## Dangers / Risks`
Potential issues, security vulnerabilities, or anti-patterns.
- Example:
  - Risk of SQL injection if inputs are not sanitized
  - Middleware order can cause security issues
  - Lack of rate limiting can lead to DoS attacks

### 1.7 `## Examples`
Code examples demonstrating proper usage.

## 4. Validation

The orchestrator will validate that all README.md files in `knowledge_base/technologies/` contain:
- All 7 mandatory sections
- All 7 additional aspects
- Valid YAML frontmatter
- Machine-readable format
