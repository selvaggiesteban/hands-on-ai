# Change Management

## Change Prioritization

When applying edits, work according to this priority:

### 1. High Priority
- **Critical security risks**: Solve critical risks in security
- **Critical architecture issues**: Address fundamental architectural problems
- **Critical testing gaps**: Fix critical testing deficiencies

### 2. Medium Priority
- Complete pending checklists for applicable technologies
- Non-critical security improvements
- Architecture optimizations
- Testing coverage improvements

### 3. Low Priority
- Style improvements
- Documentation enhancements
- Non-critical optimizations

## Application Process

When authorized to apply changes, follow this process:

### 1. Review
Review the **implementation guide** for the corresponding technology.

### 2. Apply Changes with Compliance
Apply changes **respecting compliance** with:
- Security checklist
- Architecture checklist
- Testing checklist
- Documentation checklist
- Performance checklist

### 3. Quality Control
Repeat the **deliverable quality control** after each change.

### 4. Verification
Repeat the **comparison with documentation / original prompt** and acceptance criteria.

### 5. Update Scores
Update the **final score by categories** (Security, Architecture, Testing, Documentation, Performance, SEO).

### 6. Document Changes
Document all changes made:
- What was changed
- Why it was changed
- Impact of the change
- Verification performed
